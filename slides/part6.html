Webhook 測試對話 實務應用

🔗 設定 LINE Bot Webhook

1

前往 LINE Developers Console

開啟 developers.line.biz

2

選擇你的 Channel

進入「Messaging API」頁籤

3

設定 Webhook URL

找到「Webhook settings」，點擊「Edit」

貼上剛才複製的 GAS 網址

4

驗證連線

點擊「Verify」，看到「Success」表示連線成功！

✅ 開啟 Webhook 與關閉自動回覆

開啟 Use webhook

將「Use webhook」切換到 Enabled（綠色狀態）

✓ 這樣 LINE 才會把訊息轉發到你的 GAS

關閉 Auto-reply

找到「Auto-reply messages」，點擊「Edit」

將「Auto-reply」選項關閉

⚠ 否則使用者會同時收到兩則訊息

📱 加入 Bot 好友

現在讓我們測試你的 LINE Bot！

  1. 在 LINE Developers Console 的「Messaging API」頁籤
  2. 找到 Bot 的 QR Code
  3. 用手機 LINE App 掃描 QR Code
  4. 或複製 Bot ID 搜尋加入好友
Bot ID 格式：@xxx（有 @ 符號開頭）

💬 測試對話

加入好友後，試著傳送訊息：

測試知識庫問題

問一個知識庫中有的問題：

「營業時間是？」

AI 應該會回覆正確答案！

測試知識庫以外問題

問一個知識庫中沒有的問題：

「今天天氣如何？」

AI 會先說「目前知識庫無相關資料，不過...」然後回答

📸 LINE 對話測試畫面

恭喜！你的 LINE Bot 已經成功運作了！🎉

❓ 常見問題排解

Bot 沒有回應？

  • 檢查 Webhook URL 結尾是否為 /exec
  • 確認「Use webhook」已開啟
  • 確認「Auto-reply」已關閉

回應很慢？

  • AI API 呼叫需要 2-5 秒，這是正常的
  • 可以嘗試切換到 Flash 模型（較快）

錯誤訊息？

  • 檢查 API Key 是否正確
  • 檢查試算表 ID 是否正確
  • 確認試算表有分享權限

想修改設定？

  • 回到生成器重新生成程式碼
  • 在 GAS 中貼上新程式碼
  • 重新部署（新版本）

📊 進階應用：更新知識庫

知識庫是活的！你可以隨時更新：

新增問答

在試算表中新增一列，填入問題和答案即可。

✓ 不需要重新部署程式碼！

修改答案

直接在試算表中編輯，下次問答就會生效。

✓ 即時更新，無縫接軌

多人協作

分享試算表給團隊成員，大家一起維護知識庫。

✓ 專業分工，效率加倍
