🚀 快速開始（3 步驟）
════════════════════════════════════════════════════════════════

第一次使用：

  1. 雙擊執行 setup_venv.bat（建立虛擬環境）
  2. 雙擊執行 start.bat（啟動應用程式）
  3. 瀏覽器開啟 http://localhost:8806

之後每次使用：

  1. 雙擊執行 start.bat
  2. 瀏覽器開啟 http://localhost:8806

⚠️ 注意事項
════════════════════════════════════════════════════════════════

• API Key 安全
  - 不要將 API Key 分享給他人
  - 不要將包含 API Key 的程式碼上傳到公開網站

• 費用控制
  - Gemini 有免費額度（每分鐘 15 次請求）
  - OpenAI 需付費使用，請注意用量

• 虛擬環境
  - 建議使用虛擬環境，避免套件衝突
  - 不要刪除 venv/ 資料夾

• 程式碼備份
  - 生成的 GAS 程式碼建議另外備份
  - 可在 Google Apps Script 中管理多個版本


📞 疑難排解
════════════════════════════════════════════════════════════════

問題 1：啟動時出現「找不到虛擬環境」
解決：執行 setup_venv.bat 重新建立虛擬環境

問題 2：網頁無法開啟
解決：檢查 8806 埠是否被佔用，或改用其他埠

問題 3：Bot 沒有回應
解決：
  1. 檢查 GAS 執行記錄
  2. 確認 API Key 有效
  3. 確認 Webhook 設定正確

問題 4：生成的程式碼有錯誤
解決：檢查所有必填欄位是否正確填寫

更多問題？
  → 查看 README.md 的「常見問題」章節
  → 查看 使用流程.md 的「常見問題排解」章節
