# LINE Bot GAS 生成器 - 完整使用流程

## 📌 流程總覽

```
首次使用
   ↓
建立虛擬環境 (setup_venv.bat)
   ↓
之後每次使用
   ↓
啟動應用程式 (start.bat)
   ↓
開啟網頁 (http://localhost:8806)
   ↓
填寫設定表單
   ↓
生成 GAS 程式碼
   ↓
複製程式碼
   ↓
部署到 Google Apps Script
   ↓
設定 LINE Bot Webhook
   ↓
完成！開始使用
```

---

## 🔧 步驟一：首次安裝（只需執行一次）

### Windows 使用者

1. 雙擊執行 `setup_venv.bat`
2. 等待安裝完成（約 1-2 分鐘）
3. 看到「✓ 安裝完成！」訊息後按任意鍵關閉

### macOS / Linux 使用者

```bash
# 1. 給予執行權限
chmod +x setup_venv.sh

# 2. 執行安裝腳本
./setup_venv.sh
```

---

## 🚀 步驟二：啟動應用程式

### Windows 使用者

雙擊執行 `start.bat`

### macOS / Linux 使用者

```bash
./start.sh
```

### 啟動成功畫面


在瀏覽器輸入以下任一網址：

- `http://localhost:8806`
- `http://127.0.0.1:8806`

## 📝 步驟四：填寫設定表單

### 4.1 系統基本設定（必填）

| 欄位 | 說明 | 範例 |
|------|------|------|
| LINE Channel Access Token | 從 LINE Developers 取得 | `xxxxx...` |
| Google 試算表 ID | 從試算表網址取得 | `1BxiMVs...` |
| 知識庫工作表名稱 | 試算表中的工作表名稱 | `工作表1` |

### 4.2 選擇 AI 服務（二選一）

#### 選項 A：Google Gemini

- **API Key**：從 [Google AI Studio](https://aistudio.google.com/apikey) 取得
- **模型選擇**：
  - `Gemini 2.5 Flash`：快速、經濟（推薦）
  - `Gemini 2.5 Pro`：進階、強大

#### 選項 B：OpenAI GPT

- **API Key**：從 [OpenAI Platform](https://platform.openai.com/api-keys) 取得
- **模型選擇**：
  - `GPT-4.0`：標準版
  - `GPT-4.1`：進階版
  - `GPT-5.0`：最新版

### 4.3 AI 回應風格設定

| 設定項目 | 可選選項 |
|----------|----------|
| 語氣風格 | 友善自然、專業學術、輕鬆幽默、簡短扼要、熱情鼓勵 |
| 角色定位 | AI 助理、專業客服、學習助教、生活顧問（可自訂） |
| 回答語言 | 繁體中文、簡體中文、English、日本語 |
| 回答格式 | 自然段落、條列式、Markdown、簡短扼要 |

### 4.4 知識庫設定

| 設定項目 | 說明 |
|----------|------|
| 知識庫優先級 | 高（優先使用）、嚴格（只使用）、平衡（混合） |
| 無資料提示 | 找不到資料時的開頭文字 |

### 4.5 進階參數調整

| 參數 | 範圍 | 說明 |
|------|------|------|
| Temperature | 0.0 - 1.0 | 創意度（0=保守，1=創意） |
| Max Tokens | 1024/2048/4096 | 回應長度限制 |

## ✅ 步驟五：生成程式碼

1. 確認所有必填欄位已填寫
2. 點選「🚀 生成 GAS 程式碼」按鈕
3. 等待 1-2 秒生成完成
4. 程式碼會顯示在下方的文字框中

---

## 📋 步驟六：複製並部署程式碼

### 6.1 複製程式碼

點選「📋 複製程式碼」按鈕，按鈕會變成「✅ 已複製！」

### 6.2 前往 Google Apps Script

1. 開啟 [Google Apps Script](https://script.google.com)
2. 點選「新專案」
3. 刪除預設的程式碼
4. 貼上剛才複製的程式碼
5. 點選「儲存」（可為專案命名，例如「LINE Bot 主程式」）

### 6.3 部署為網路應用程式

1. 點選「部署」→「新增部署作業」
2. 點選「選取類型」→ 選擇「網路應用程式」
3. 填寫說明（例如：「LINE Bot v1.0」）
4. 設定：
   - **執行身分**：選擇「我」
   - **具有存取權的使用者**：選擇「所有人」
5. 點選「部署」
6. 授權存取（首次需要）
7. **複製網路應用程式網址**（非常重要！）

範例網址格式：
```
https://script.google.com/macros/s/AKfycby.../exec
```

---

## 🤖 步驟七：設定 LINE Bot Webhook

### 7.1 前往 LINE Developers Console

1. 開啟 [LINE Developers](https://developers.line.biz/)
2. 選擇你的 Provider
3. 選擇你的 Channel（Messaging API）

### 7.2 設定 Webhook URL

1. 找到「Messaging API」頁籤
2. 滾動到「Webhook settings」區塊
3. 點選「Edit」
4. 貼上剛才複製的 GAS 網路應用程式網址
5. 點選「Update」
6. 點選「Verify」測試連線（應顯示 Success）

### 7.3 啟用 Webhook

1. 在「Use webhook」右側
2. 將開關切換到「Enabled」（綠色）

### 7.4 關閉自動回覆（重要！）

1. 找到「Auto-reply messages」區塊
2. 點選「Edit」
3. 將「Auto-reply」關閉
4. 將「Greeting message」也建議關閉（可選）

---

## 🎉 步驟八：測試 LINE Bot

### 8.1 加入 LINE Bot 好友

1. 在 LINE Developers Console 找到「Messaging API」頁籤
2. 掃描 QR Code 或複製 Bot ID
3. 在 LINE App 中加入好友

### 8.2 測試對話

試著傳送訊息給 Bot：

- **測試知識庫問題**：傳送試算表中有的問題
- **測試一般問題**：傳送試算表中沒有的問題
- **觀察回應風格**：確認語氣是否符合設定

### 8.3 檢查錯誤（如果沒有回應）

如果 Bot 沒有回應，請檢查：

1. **GAS Log**：在 Apps Script 點選「執行作業」查看錯誤
2. **API Key**：確認 Gemini 或 OpenAI API Key 是否有效
3. **試算表權限**：確認 GAS 有權限讀取試算表
4. **Webhook**：確認 LINE Bot Webhook 設定正確且已啟用

## 🔄 如需修改設定

### 方法一：重新生成程式碼

1. 回到生成器網頁（`http://localhost:8806`）
2. 點選「🔄 重新設定」按鈕
3. 修改想要變更的設定
4. 重新生成程式碼
5. 回到 GAS，貼上新的程式碼
6. 重新部署（或點選「部署」→「管理部署作業」→「編輯」→「版本：新版本」）

### 方法二：直接修改 GAS 程式碼

如果只是小修改（例如調整 Temperature），可以直接在 GAS 中修改程式碼開頭的設定區：

```javascript
// --- AI 回應風格設定 ---
const TEMPERATURE = 0.5;  // 直接修改這裡
```

修改後記得儲存！

## 💡 進階技巧

### 1. 知識庫管理

**試算表格式範例**：

| 問題 | 答案 |
|------|------|
| 營業時間是？ | 我們的營業時間是週一至週五 9:00-18:00 |
| 如何聯絡客服？ | 請撥打客服專線：02-1234-5678 |
| 退貨政策？ | 商品到貨 7 日內可無條件退貨 |

### 2. 多版本管理

建議為不同版本命名：
- `LINE Bot v1.0 - 友善版`
- `LINE Bot v2.0 - 專業版`
- `LINE Bot v3.0 - 幽默版`

在 GAS 中可以同時保留多個部署版本。

### 3. 效能優化建議

- **Gemini 2.5 Flash**：回應速度快，適合即時對話
- **Gemini 2.5 Pro**：回應品質高，適合複雜問題
- **降低 Temperature**：回答更穩定一致
- **限制 Max Tokens**：減少 API 費用

## ❓ 常見問題排解

### Q1: 啟動時出現「找不到 Python」

**解決方法**：
1. 確認已安裝 Python 3.7+
2. 檢查 Python 是否已加入環境變數 PATH
3. 重新安裝 Python，安裝時勾選「Add to PATH」

### Q2: 生成的程式碼沒有效果

**檢查清單**：
- [ ] API Key 是否正確且有效
- [ ] 試算表 ID 是否正確
- [ ] 工作表名稱是否與實際相符
- [ ] LINE Channel Access Token 是否正確
- [ ] 是否已正確部署 GAS

### Q3: Bot 回應很慢

**可能原因**：
- Gemini/OpenAI API 回應時間（通常 2-5 秒）
- 試算表資料量過大
- 網路連線速度

**優化建議**：
- 使用 Gemini 2.5 Flash（更快）
- 精簡知識庫內容
- 降低 Max Tokens

### Q4: 想切換 AI 服務（Gemini ↔ OpenAI）

重新在生成器中選擇不同的 AI 服務，生成新的程式碼並重新部署即可。
